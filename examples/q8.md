# Background
Structured output tasks require models to return data in a specific format. This can be achieved through various methods, including JSON mode or constrained decoding. However, existing structured output benchmarks for LLMs contain major errors in the provided "ground-truth" outputs [10].

# Key Findings
Gemini 2.5 Flash Preview (Reasoning) has image input support while Llama 3.3 Instruct 70B does not [1]. Gemini 2.5 Flash-Lite leads on benchmark performance compared to Llama 3.3 70B Instruct [2]. # Open Questions / Gaps
There are several frameworks and tools that compare models on real structured output workflows, including Promptify [5], llama-cpp-agent [6], zod-gpt [7], and ollama-swift [8]. However, the impact of structured outputs on LLM performance, particularly on tasks that require open-ended responses, is still an open question [12].
